# Ilex frontend discipline

Architecture in [`docs/architecture.md`](../../../docs/architecture.md). Decisions in [`docs/decisions.md`](../../../docs/decisions.md). General spec in [`docs/specs/SPEC.md`](../../../docs/specs/SPEC.md). This skill is the rule set.

## Hard invariants

CI fails on violation:

1. **No hand-written API types.** `src/api/generated/` is regenerated from the server's OpenAPI 3.1 schema via `openapi-typescript`. Never hand-edit; never parallel-define request/response shapes elsewhere.
2. **No bare `fetch` / `axios` outside the data layer.** All JSON HTTP goes through generated client functions called from `src/data/`. The only documented exception is `src/utils/csv-export.ts` (anchor-driven CSV downloads — see SPEC §2.5).
3. **No `as any` outside `src/api/generated/`.** Strict TypeScript. If a generated type is wrong, fix the server's OpenAPI annotations, not the FE.
4. **Money and quantity are strings end-to-end** (matching BE's `numeric(14, 4)`). Arithmetic via `Decimal.js`. Never JS `number` math on money or qty paths.
5. **No `NumberInput` in money/qty fields.** Mantine's `NumberInput` returns `number` and breaks invariant 4. Use `<DecimalInput>` (`src/components/DecimalInput.tsx`). `NumberInput` is allowed only for integer-only contexts (page size, `expiring_within` days).
6. **Routes do not call APIs directly.** Features call data hooks; data hooks call the generated client. Cross-layer imports upward are forbidden.
7. **Cross-owner access returns 404 from BE** (BE-D4). UI surfaces "Not found" without distinguishing from never-existed.
8. **Terminal mutations never optimistic** (PO receive, SO commit, SO void, recall, un-recall, write-off, manual batch, products import). Invalidate + refetch on success. Optimistic updates are allowed only on trivial PATCHes touching a single row's display fields (e.g., product name).

## Layer rules

Data flows Routes → Features → Data → API client. No imports upward.

### Routes (`src/routes/*.tsx`)

- One file per page. Declares path + page component.
- Optional TanStack Query loader for pre-fetch; nothing else.
- No business logic, no inline forms, no direct API client calls.

### Features (`src/features/{domain}/`)

- Page components compose UI for a flow.
- Feature components are domain-specific UI (PO line editor, FEFO preview, recall toggle).
- Forms validate locally and submit via mutation hooks.
- May import from `src/data/{domain}/` and `src/components/`. May NOT import `src/api/generated/` directly.
- Mantine + Tailwind only. Tailwind for one-offs; no raw CSS modules outside `src/theme/`.

### Data (`src/data/{domain}/`)

- Files: `queries.ts`, `mutations.ts`, `keys.ts` per domain.
- Query hooks: `use{Noun}Query`, `use{Noun}List`. Wrap TanStack `useQuery` / `useInfiniteQuery`.
- Mutation hooks: `use{Verb}{Noun}Mutation`. Wrap TanStack `useMutation`. Mints `Idempotency-Key` (UUIDv7) for the BE-required terminal endpoints. Maps 409 on terminal mutations to refetch + toast (per SPEC §2.5).
- Cache keys centralized in `keys.ts`. Mutations invalidate the affected lists/details on success.
- May import from `src/api/`. May NOT render UI.

### API client (`src/api/`)

- `generated/` — regenerated by `npm run generate:api`. Never hand-edited. Committed.
- `client.ts` — thin wrapper adding `credentials: 'include'`, CSRF (`X-CSRFToken` from `csrftoken` cookie), `Idempotency-Key` plumbing, and 4xx error envelope normalization.

## Errors

The data layer normalizes the BE 4xx envelope `{ error: ErrorCode, detail?: string, fields?: Record<string, string> }` into a typed `ApiError`. Features render:
- `fields` → field-level form messages
- `detail` → toast on non-form errors
- 409 on terminal mutations → refetch the affected detail + toast `"This {resource} has already been {received|committed|voided|...} elsewhere."`
- 404 on detail routes → "Not found" (do not distinguish cross-owner from never-existed)

## File layout per domain

```
src/
├── routes/
│   └── {domain}.{action}.tsx        # one per page
├── features/{domain}/
│   ├── {Page}.tsx                   # page components
│   └── components/                  # domain-specific UI
├── data/{domain}/
│   ├── queries.ts
│   ├── mutations.ts
│   └── keys.ts
├── api/
│   ├── generated/                   # regenerated; do not edit
│   └── client.ts
├── components/                      # shared UI (DecimalInput, ConfirmModal, ...)
├── hooks/                           # UI-only hooks
├── stores/                          # Zustand stores (floor mode, agent panel, ⌘K)
├── utils/                           # money.ts, qty.ts, csv-export.ts
└── theme/
```

## Naming

| Item | Convention |
|---|---|
| Route file | `{domain}.{action}.tsx` (e.g. `purchase-orders.new.tsx`) |
| Page component | `{Domain}{Action}Page.tsx` |
| Feature component | `{Noun}{Role}.tsx` (e.g. `FefoPreview.tsx`) |
| Query hook | `use{Noun}Query` / `use{Noun}List` |
| Mutation hook | `use{Verb}{Noun}Mutation` |
| Cache key | `{domain}Keys.{noun}({...args})` |
| Test file | `{Subject}.test.tsx` (colocated) |
| Generated module | `src/api/generated/{operation}.ts` |

## CI gates

- `npm test` all green
- `npm run typecheck` (strict) passes
- `npm run lint` passes
- `npm run generate:api -- --check` returns no diff against the server's OpenAPI
- `grep -RE "(fetch|axios)\(" src/ --include='*.ts' --include='*.tsx' | grep -vE "src/data/|src/api/|src/utils/csv-export\.ts"` returns nothing
- `grep -RE "as any" src/ --include='*.ts' --include='*.tsx' | grep -v "src/api/generated"` returns nothing
- `grep -RE "from ['\"].*api/generated" src/features/ src/routes/` returns nothing — features call data hooks, not the generated client
- `grep -RE "\bNumberInput\b" src/features/` returns nothing outside the integer-only allowlist
- No `console.log` left behind (`grep -RE "console\.(log|debug)" src/ --include='*.ts' --include='*.tsx'` returns nothing in committed code)
